環境変数

printenv : 設定されている環境変数を表示できる

export LESS=’–no-init’ : 環境変数を設定（lessコマンド終了時に画面を消去しない)

環境変数は~/.bash_profileから読み込まれるが、この中には~/.bashrcを読み込むように書かれている。profileはログイン時にしか読み込まれないが、.bashrcはbash起動のために読み込まれるため、普通は後者に書き込んだほうが良い。

テキスト処理

sort 行を並べ替える
-rは逆順 -nは数値順
uniq 重複行を取り除く(連続した行でないと取り除けないのでsortで並べ替えてから実行する。
-cで重複行を数える
cut 入力の一部を切り出す
-d <区切り文字> -f <フィールド番号> [<ファイル名>]
cat ファイル内容をそのまま表示する

grep

grep [オプション] <検索パターン> <ファイル名>

たとえば、

grep bash ~/.bashrc

行番号をつける時は、

grep -n  bash ~/.bashrc

オプションがないときは大文字小文字を区別する。-iをつけると区別しなくなる。

grep -i Bash ~/.bashrc

-vをつけると含まない行を出力する

grep -v bash ~/.bashrc

正規表現

正規表現を使うならシングルクォートで囲う。bashで始まる行を抽出したいなら、

grep '^bash' ~/.bashrc

※クォートで囲まないとシェルが展開してしまう。これはカレントディレクトリ内にあるbashを含むファイルに展開されてしまう。

grep bash* ~/.bashrc

任意の一文字は.(ドット)で表す。

grep 'b.sh' ~/.bashrc

ドットそのものをマッチさせたい場合は.のまえにバックスラッシュを書く。

grep 'bash\.' ~/.bashrc

特定の文字を指定するときは[]で囲む。[a-zA-Z]は大文字小文字全てにマッチ。[1-4]は1,2,3,4はマッチ。[^13]は1,3以外にマッチ。

grep 'b[ab]sh' ~/.bashrc

行頭の文字列を指定する場合は^を、行末を指定する場合は$をつける

grep '^bash' ~/.bashrc
grep 'bash$' ~/.bashrc

空行は^$で表すことができる。

grep -v '^$' ~/.bashrc

で空行を除くことができる。

正規表現において、*は直前の文字の0回以上の繰り返しを表す。例えば go*gle は

ggle gogle google gooogle goooogle

にマッチする。この*はbashにおけるワイルドカードとは異なる。ワイルドカードでの*は0文字以上の任意の文字列を意味する。同じ意味を正規表現で表すためには .* を用いる。

拡張正規表現

grepで-Eオプションをつけると上記以外の正規表現も使えるようになる。

+は直前の文字の1回以上の繰り返しを表す。

grep -E 'ba+h' ~/.bashrc

これはbah baah baaah …にマッチし、bhにはマッチしない。

繰り返し回数を指定したい場合は{m,n}を使う。基本正規表現で使いたい場合は\{m,n\}とする。

grep -E 'ba{1,3}h' ~/.bashrc

はbah baah baaahにマッチする。例えば、郵便番号は[0-9]{3}-[0-9]{4}で、これらはちょうど3回, 4回の繰り返しを意味する。

()は正規表現をグループ化することができる。 (bash){2,} はbashが2回以上繰り返される場合にマッチ。

| はorの意味で、abc|def|xyzはabc, def, xyzのいずれかにマッチする。

sed-非対話型エディタ

sed [Option] <script> <File>

スクリプトはコマンドとアドレス、フラグからなる。

sed 1,5d sample.txt #1,5は1行目と5行目を指すアドレス、dは削除コマンド
sed 's/aaa/bbb/g' sample.txt #sは置換コマンド、aaa/bbbは置換前後の文字列、gはフラグで各行に現れた全ての該当文字を削除する

$は最終行を意味する

sed 3,$d sample.txt

は3行目から最終行まで削除する。

置換が発生した行だけ表示する

sed -n 's/!//gp' sample.txt

コマンド直後の/は区切り文字になる。例えば、

sed 's!bear!/bear/!g' sample.txt

は!が区切り文字で、bearを/bear/に置換する。

awk

awk <script> <file>
awk 'pattern {action}' file

スクリプトはパターンとアクションからなる。パターンはアクションを実行するかどうか、アクションはテキストの抽出、置換、削除などをパターンにマッチした場合に実行。パターンを省略するとすべてのレコード（行のこと）に対してアクションを実行する。

第2・第3フィールドを表示する

awk '{print $2,$3}' sample.txt

レコード全体は $0 で指定できる。変数は,で区切らないとスペースが入らない。 $NF はレコードの最後のフィールドが表示される。 $(NF-1) は最後から2つ目。

第9フィールドがcpで始まる場合にアクションを実行する場合は

awk '$9 ~ /^cp/ {print $5, $9}' sample.txt

~は正規表現の比較を意味し、正規表現のパターンはスラッシュ/で囲む。比較対象($9)は省略しても良い。

awk '/^cp/ {print $5, $9}' sample.txt

この場合はレコード全体を対象とする。

区切り文字を,に指定して、最終フィールドの総和を計算したいなら、

awk -F, '{sum += $NF} END{print sum}' sample.csv

ENDはENDブロックというパターン。ENDブロック内のアクションは最後に実行される。NRは行数と同じであり、 sum/NRは平均を意味する。スクリプトをsum.awkとして保存することもでき、

awk -F, -f sum.awk score.csv

sum.awk
      {sum +=$NF} END{print sum}

のように実行する。
