githubのssh認証について

 Aug 14, 2020      Note 

githubを使用するときのパスワード入力の手間を省くことができます。

https://docs.github.com/ja/github/authenticating-to-github/connecting-to-github-with-ssh

が参考になりますが、やったことは以下の通りです。

  1. まず、sshキーが存在するかどうかを以下のコマンドで確認する。

    ls -al ~/.ssh

公開鍵のファイル名は以下のいずれか。

  • id_rsa.pub

  • id_ecdsa.pub

  • id_ed25519.pub

  • sshキーがない場合、以下を実行して新しいキーを作る

    ssh-keygen -t rsa -b 4096 -C "mail@example.com"

  • Enter a file in which to save the key、Enter passphraseが表示されるが空白のままEnterを押す（パスフレーズを使わない）

  • 以下を実行してsshキーをクリップボードにコピーする

    clip < ~/.ssh/id_rsa.pub

  • githubでNew SSH keyを選択し、Titleを決めて、4.でコピーしたキーを貼りつける。

  • Add SSH keyを押す

  • Bash上で以下を実行する

    ssh -T git@github.com

    以下のような警告が出てきた場合は、yesを押す

    > The authenticity of host 'github.com (IP ADDRESS)' can't be established.
> RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspRomTxdCARLviKw6E5SY8.
> Are you sure you want to continue connecting (yes/no)?

    2行目のfingerprintはsshキーの指紋で、以下で確認できる。

    ssh-keygen -lf ~/.ssh/id_rsa

    -lvfとすれば指紋のAscii artを見ることができる。

  • yesを押すと以下のメッセージが表示され、sshキーによる認証ができるようになる。

    > Hi username! You've successfully authenticated, but GitHub does not
> provide shell access.

  • ssh認証によるpushやgit clone git@github.com:USERNAME/REPOSITORYのようにcloneが可能になる。

  • 既にhttpsでcloneしているものは以下でurl確認して

$ git remote -v
origin  https://github.com/foo/foo.git.io.git (fetch)
origin  https://github.com/foo/foo.git.io.git (push)

次のコマンドでhttps->git@、.comの後の/を:に変える

git remote set-url origin git@github.com:foo/foo.git.io.git

もう一度最初のコマンドを実行して内容が反映されているか確認する。submodule分は.gitmoduleファイルと.git/configを書き換える。

git config -e

とするとエディタが起動するのでhttpsからgit@、/から:に変える

これで、パスワードの入力が不要になると思います。なお、ssh-agentを使うと秘密鍵に設定されたパスフレーズを繰り返し入力しなくて済むようになるみたいですが、ここでは省略しています。
